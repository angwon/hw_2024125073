## 1. 서론

### 1.1 목적 및 범위
이 문서는 사용자 신고 데이터를 기반으로 동네 안전 지도를 생성하고, AI 분석을 통해 안전 경로 추천 및 실시간 알림을 제공하는 **'세이프맵 (SafeMap)'** 프로젝트의 요구사항을 조사하고 정의하는 문서이다.
이 문서는 시스템의 핵심인 위험 요소 신고, AI 기반 위험도 분석, 안전 경로 추천 기능 등을 포함한 기능적 요구사항과, 시스템 운영 및 인터페이스에 요구되는 사항들을 정의한다.

### 1.2 용어 정의
본 문서의 이해를 돕기 위해 사용된 모든 용어 및 약어를 설명하고 정의합니다.

| 용어 | 설명 |
|---|---|
| **RESTful API** | 두 컴퓨터 시스템이 인터넷을 통해 정보를 안전하게 교환하기 위해 사용하는 인터페이스 디자인 스타일 |
| **다익스트라 알고리즘** | 여러 노드(지점)가 연결된 그래프에서 특정 출발점에서 다른 모든 지점까지 가는 최단 경로를 찾는 알고리즘 |
| **인터페이스** | 서로 다른 존재들이 정보를 주고받기 위해 지켜야 하는 규칙이나 연결 장치 |

### 2.2 기능 분류 및 설명

#### 2.2.1 UseCase Description

> **📌 UC_01: 회원가입**
> * **Importance Level:** high
> * **Primary Actor:** 사용자
> * **Use Case Type:** Concrete, Essential

* **Brief Description:** 사용자가 서비스를 이용하기 위해 본인의 식별 정보 및 위치 정보 활용 동의를 제공하고 시스템 계정을 생성한다.
* **Stakeholders and Interests:**
  * **사용자:** 개인정보를 안전하게 등록하고 서비스 권한을 획득하고자 함.
  * **관리자:** 유효한 사용자 데이터를 확보하고 서비스 약관 동의를 법적으로 증빙함.
* **Trigger:** 사용자가 초기 화면에서 '회원가입' 버튼을 클릭함.
* **Relationships:**
  * **Association:** 사용자 (Actor)
  * **Include:** 본인 인증, 위치 정보 동의
  * **Extend:** 프로필 이미지 등록
  * **Generalization:** 이 UseCase는 이메일 회원가입과 소셜 계정 회원가입으로 구체화됨
* **Normal Flow of Events:**
  1. 사용자가 '회원가입' 버튼을 누른다.
  2. 시스템은 이용약관 및 위치 기반 서비스 이용 동의서를 화면에 제시한다.
  3. 사용자가 필수 항목에 동의하고 '다음'을 선택한다.
  4. 시스템은 사용자 정보 입력창(이메일, 비밀번호, 닉네임 등)을 출력한다.
  5. 사용자가 정보를 입력하고 가입 요청을 한다.
  6. 시스템은 이메일 중복 여부 및 비밀번호 정책 준수 여부를 확인한다.
  7. 시스템은 사용자 정보를 데이터베이스에 저장하고 계정을 생성한다.
  8. 가입 완료 메시지를 출력하며 로그인 화면으로 안내한다.
* **Subflows:**
  * **S1.** 비밀번호 정책 확인: 영문, 숫자, 특수문자 조합 8자리 이상 여부를 실시간으로 체크함.
  * **S2.** 위치 정보 동의: 시스템 정의서(HW1)의 필수 제약에 따라 위치 정보 동의 프로세스를 반드시 거침.
* **Alternate / Exceptional Flows:**
  * **A1.** 소셜 계정 가입: 3번 단계에서 사용자가 카카오/구글 가입을 선택할 경우 외부 인증 서버로 이동한다.
  * **E1.** 중복 이메일: 이미 가입된 이메일일 경우 "사용 중인 이메일입니다" 메시지를 출력하고 4번으로 복귀한다.
  * **E2.** 필수 약관 미동의: 동의하지 않을 경우 가입이 불가함을 알리고 메인 화면으로 이동한다.

> **📌 UC_02: 로그인**
> * **Importance Level:** high
> * **Primary Actor:** 사용자
> * **Use Case Type:** Concrete, Essential

* **Brief Description:** 등록된 사용자가 본인의 계정 정보를 입력하여 시스템에 접속하고, 개인화된 서비스를 이용할 수 있는 권한을 얻는다.
* **Stakeholders and Interests:**
  * **사용자:** 안전하게 자신의 계정에 접속하여 저장된 데이터와 개인 설정을 불러오고자 함.
  * **관리자:** 허가된 사용자만 시스템을 이용하게 하여 허위 신고 리스크를 관리하고 보안을 유지함.
* **Trigger:** 사용자가 앱을 실행했거나, 로그인이 필요한 기능을 클릭했을 때 로그인 화면이 호출됨.
* **Relationships:**
  * **Association:** 사용자 (Actor)
  * **Include:** 사용자 인증
  * **Extend:** 비밀번호 찾기
  * **Generalization:** 이 UseCase는 이메일 로그인과 소셜 로그인으로 일반화됨.
* **Normal Flow of Events:**
  1. 사용자가 로그인 화면에서 식별 정보(이메일)와 비밀번호를 입력한다.
  2. 사용자가 '로그인' 버튼을 클릭한다.
  3. 시스템은 입력된 정보가 DB에 저장된 해시값과 일치하는지 확인한다.
  4. 정보가 일치하면 시스템은 사용자 고유 토큰을 생성하여 세션을 수립한다.
  5. 사용자의 신뢰도 점수 및 이전 설정 데이터를 로드한다.
  6. 로그인이 성공했음을 알리고 메인 위험 지도 화면으로 이동한다.
* **Subflows:**
  * **S1.** HTTPS 암호화: HW3 요구사항에 따라 아이디/비밀번호 전송 시 모든 데이터는 HTTPS로 암호화됨.
  * **S2.** 자동 로그인: 사용자가 '로그인 유지'를 선택한 경우, 로컬 기기에 암호화된 토큰을 저장함.
* **Alternate / Exceptional Flows:**
  * **A1.** 소셜 로그인: 사용자가 카카오/구글 로그인을 선택할 경우, 외부 인증 서버의 성공 응답을 받아 로그인을 완료함.
  * **E1.** 계정 정보 불일치: 이메일 혹은 비밀번호가 틀린 경우 "정보가 일치하지 않습니다" 메시지를 출력하고 1번으로 돌아감.
  * **E2.** 계정 잠금: 연속 5회 이상 로그인 실패 시, 보안을 위해 일정 시간 계정을 잠금 처리함.

> **📌 UC_03: 실시간 위험 지도 조회**
> * **Importance Level:** high
> * **Primary Actor:** 사용자
> * **Use Case Type:** Concrete, Essential

* **Brief Description:** 사용자가 현재 위치 또는 특정 지역의 위험 정보를 시각화된 지도를 통해 실시간으로 확인한다.
* **Stakeholders and Interests:**
  * **사용자:** 주변의 위험 요소를 한눈에 파악하여 사고를 예방하고 안전 의식을 고취함.
  * **관리자:** 사용자들에게 최신 분석 데이터를 제공하여 서비스의 신뢰도를 유지함.
* **Trigger:** 사용자가 로그인을 완료하고 메인 화면에 진입하거나, 지도 화면을 새로고침함.
* **Relationships:**
  * **Association:** 사용자 (Actor)
  * **Include:** 위치 정보 획득, 위험 데이터로드
  * **Extend:** 위험 요소 상세 조회
  * **Generalization:** 없음.
* **Normal Flow of Events:**
  1. 시스템은 사용자의 현재 GPS 위치를 기반으로 지도 화면을 초기화한다.
  2. 시스템은 해당 좌표 반경 내의 위험 신고 데이터 및 AI 분석 지수를 서버에 요청한다.
  3. 서버는 실시간으로 검증된 위험 정보와 공공데이터를 결합하여 응답한다.
  4. 시스템은 지도 위에 위험도에 따른 색상과 위험 유형별 아이콘을 렌더링한다.
  5. 사용자는 지도를 드래그하거나 확대/축소하여 다른 지역의 위험도를 탐색한다.
  6. 지도상의 아이콘을 클릭하여 해당 지점의 간략한 정보를 확인한다.
* **Subflows:**
  * **S1.** 위험도 시각화: 동일 위치 신고 횟수와 범죄 통계를 기반으로 위험도를 3단계(안전, 주의, 위험)로 구분하여 색상을 출력함.
  * **S2.** 데이터 캐싱: 네트워크 트래픽 절감을 위해 일정 시간 내의 데이터는 로컬 캐시를 활용함.
* **Alternate / Exceptional Flows:**
  * **A1.** 검색을 통한 위치 이동: 사용자가 주소 검색창에 특정 위치를 입력하면 해당 지역으로 지도를 이동시키고 데이터를 다시 로드함.
  * **E1.** 네트워크 연결 없음: HW3 제약사항에 따라 "네트워크 연결을 확인하세요" 메시지를 출력하고 빈 지도를 보여줌.
  * **E2.** 데이터 없음: 신고 데이터가 없는 청정 구역의 경우 "위험 요소가 발견되지 않은 안전한 지역입니다" 문구를 표시함.

> **📌 UC_04: 위험 요소 상세 정보 열람**
> * **Importance Level:** medium
> * **Primary Actor:** 사용자
> * **Use Case Type:** Concrete, Essential

* **Brief Description:** 사용자가 지도상의 위험 아이콘을 클릭하여 해당 위험 요소의 구체적인 내용(사진, 위험 유형, 신고 일시, 상세 설명)을 확인한다.
* **Stakeholders and Interests:**
  * **사용자:** 특정 구역이 왜 위험한지 구체적인 사유를 파악하여 주의를 기울임.
  * **신고자:** 자신이 기여한 신고 정보가 타인에게 유용한 정보로 제공됨을 확인함.
* **Trigger:** 사용자가 실시간 위험 지도(UC-03) 상에 표시된 특정 위험 아이콘을 클릭함.
* **Relationships:**
  * **Association:** 사용자 (Actor)
  * **Include:** 없음
  * **Extend:** 신고 신뢰도 평가, 신고 내용 수정
  * **Generalization:** 없음.
* **Normal Flow of Events:**
  1. 사용자가 지도 화면에서 특정 위험 지점 아이콘을 선택한다.
  2. 시스템은 해당 신고 건의 고유 ID를 서버에 전송하여 상세 데이터를 요청한다.
  3. 서버는 해당 위험 요소의 사진, 유형, 상세 설명, 신고 시각, 현재 신뢰도 점수를 응답한다.
  4. 시스템은 하단 시트 또는 팝업창을 통해 상세 정보를 출력한다.
  5. 사용자는 정보를 확인한 후 상세 창을 닫거나, 정보의 정확성에 대해 평가(좋아요/싫어요)를 할 수 있다.
* **Subflows:**
  * **S1.** 비식별화 처리: HW1 제약사항에 따라 신고자의 개인정보(ID 등)는 마스킹 처리하여 노출함.
  * **S2.** 실시간 상태 표시: 현재 해당 위험 요소가 조치 중인지, 혹은 유효한 상태인지 정보를 표시함.
* **Alternate / Exceptional Flows:**
  * **A1.** 본인 게시물 확인: 조회한 정보가 본인이 작성한 것일 경우 '수정' 및 '삭제' 버튼을 추가로 활성화함.
  * **E1.** 데이터 삭제됨: 정보를 불러오는 사이에 신고자가 게시물을 삭제한 경우 "이미 삭제된 정보입니다" 메시지를 출력하고 지도를 새로고침함.
  * **E2.** 이미지 로딩 실패: 네트워크 문제로 사진이 뜨지 않을 경우 기본 대체 이미지(Placeholder)를 출력함.

> **📌 UC_05: 위험 요소 신고**
> * **Importance Level:** high
> * **Primary Actor:** 사용자
> * **Use Case Type:** Concrete, Essential

* **Brief Description:** 사용자가 현장에서 발견한 위험 요소(범죄 취약, 시설 파손 등)를 GPS 위치와 사진을 포함하여 실시간으로 시스템에 등록한다.
* **Stakeholders and Interests:**
  * **사용자:** 자신이 발견한 위험을 공유하여 공동체의 안전에 기여하고 신뢰도 점수를 얻고자 함.
  * **관리자:** 정확한 현장 데이터를 수집하여 위험 지도를 최신 상태로 유지하고자 함.
* **Trigger:** 사용자가 앱 하단의 '신고하기(+)' 버튼을 클릭함.
* **Relationships:**
  * **Association:** 사용자 (Actor)
  * **Include:** 위치 정보 획득, 이미지 업로드
  * **Extend:** 위치 수동 보정
  * **Generalization:** 없음.
* **Normal Flow of Events:**
  1. 사용자가 '신고하기' 버튼을 누른다.
  2. 시스템은 현재 사용자의 GPS 좌표를 자동으로 획득하여 지도에 표시한다.
  3. 사용자가 위험 카테고리를 선택한다.
  4. 사용자가 현장 사진을 촬영하거나 갤러리에서 선택하여 첨부한다.
  5. 사용자가 위험 상황에 대한 상세 설명을 입력한다.
  6. 사용자가 '제출' 버튼을 클릭한다.
  7. 시스템은 데이터의 유효성을 검사하고, 사진 내 개인정보를 비식별화 처리한다.
  8. 시스템은 신고 정보를 DB에 저장하고 사용자에게 완료 알림 및 신뢰도 포인트를 부여한다.
* **Subflows:**
  * **S1.** AI 이미지 분석: 업로드된 사진이 선택한 위험 카테고리와 일치하는지 AI가 1차적으로 판별함.
  * **S2.** 비식별화: HW2 제약사항에 따라 저장 전 사진 내의 민감 정보를 자동으로 블러 처리함.
* **Alternate / Exceptional Flows:**
  * **A1.** 위치 보정: GPS 오차가 클 경우, 사용자가 지도에서 핀을 드래그하여 정확한 위치로 수정할 수 있음.
  * **E1.** 필수 항목 누락: 사진이나 카테고리 미선택 시 "필수 정보를 입력해 주세요" 메시지를 출력하고 3번 단계로 복귀함.
  * **E2.** 허위 신고 의심: HW3 운영 정책에 따라 단시간 내 동일 위치 반복 신고 시 "중복 신고"로 분류하여 처리를 제한함.

> **📌 UC_06: 신고 내용 수정/삭제**
> * **Importance Level:** medium
> * **Primary Actor:** 사용자
> * **Use Case Type:** Concrete, Essential

* **Brief Description:** 사용자가 자신이 이전에 등록한 위험 요소 신고 게시물의 내용을 수정하거나, 더 이상 유효하지 않은 정보를 삭제한다.
* **Stakeholders and Interests:**
  * **사용자:** 잘못 입력한 정보를 바로잡거나 상황이 종료된 신고를 삭제하여 정확한 정보를 유지함.
  * **관리자:** 사용자 스스로 데이터를 정제하게 함으로써 지도 데이터의 최신성을 확보함.
* **Trigger:** 사용자가 내 활동 내역 또는 상세 정보 화면에서 본인이 작성한 게시물의 '수정' 또는 '삭제' 버튼을 클릭함.
* **Relationships:**
  * **Association:** 사용자 (Actor)
  * **Include:** 본인 여부 확인
  * **Extend:** 신뢰도 점수 재계산
  * **Generalization:** 없음.
* **Normal Flow of Events:**
  * **[수정 시]**
    1. 사용자가 본인의 신고물 상세 화면에서 '수정' 버튼을 클릭한다.
    2. 시스템은 기존에 입력된 데이터가 채워진 수정 폼을 제공한다.
    3. 사용자가 내용을 변경하고 '수정 완료'를 클릭한다.
    4. 시스템은 변경된 정보를 DB에 업데이트하고 지도에 즉시 반영한다.
  * **[삭제 시]**
    1. 사용자가 본인의 신고물 상세 화면에서 '삭제' 버튼을 클릭한다.
    2. 시스템은 삭제 확인 팝업을 출력한다.
    3. 사용자가 삭제를 확정하면 시스템은 해당 데이터를 처리한다.
    4. 시스템은 지도상에서 해당 아이콘을 제거한다.
* **Subflows:**
  * **S1.** 신뢰도 연동: 수정 시 AI 분석 결과와 일치율이 높아지면 신뢰도 점수를 추가 부여하고, 삭제 시 가입 시 부여된 기본 점수 외의 활동 점수를 회수함.
  * **S2.** 데이터 보존: 삭제된 데이터는 향후 통계 및 허위 신고 분석을 위해 서버 로그에 별도 보관함.
* **Alternate / Exceptional Flows:**
  * **A1.** 관리자에 의한 강제 삭제: 허위 신고로 판명되어 관리자가 삭제한 경우, 사용자는 수정/삭제 권한을 상실하며 알림을 받음.
  * **E1.** 권한 없음: 타인이 작성한 게시물에 대해 수정/삭제를 시도할 경우 "권한이 없습니다" 메시지를 출력하고 기능을 차단함.
  * **E2.** 수정 횟수 제한: 과도한 수정을 통한 데이터 교란을 방지하기 위해 수정 가능 횟수를 제한함.

> **📌 UC_07: 안전 경로 목적지 검색**
> * **Importance Level:** high
> * **Primary Actor:** 사용자
> * **Use Case Type:** Concrete, Essential

* **Brief Description:** 사용자가 가고자 하는 목적지를 입력하면, 시스템은 신고 데이터와 AI 분석 결과를 바탕으로 위험 지역을 회피하는 최적의 안전 경로를 탐색하여 안내한다.
* **Stakeholders and Interests:**
  * **사용자:** 범죄 취약지나 사고 발생 지역을 피해 가장 안전하게 목적지까지 이동하고자 함.
  * **관리자:** 축적된 신고 데이터를 활용해 사용자에게 실질적인 가치를 제공하고 사고 발생률을 낮춤.
* **Trigger:** 사용자가 메인 화면의 검색창에 목적지를 입력하거나, 지도에서 지점을 롱 클릭하여 '목적지로 설정'을 선택함.
* **Relationships:**
  * **Association:** 사용자 (Actor)
  * **Include:** 위치 정보 획득, 경로 탐색 알고리즘 실행
  * **Extend:** 즐겨찾기 경로 추가
  * **Generalization:** 없음.
* **Normal Flow of Events:**
  1. 사용자가 검색창에 목적지를 입력한다.
  2. 시스템은 현재 위치와 입력된 목적지를 확인한다.
  3. 시스템은 두 지점 사이의 가능한 모든 경로 후보를 탐색한다.
  4. AI 분석 모듈이 각 경로에 포함된 '위험 지수'와 '최근 신고 이력'을 가중치로 계산한다.
  5. 위험 총점이 가장 낮으면서 이동 시간이 적절한 '안전 경로'를 산출한다.
  6. 지도 위에 최단 경로와 안전 경로를 대비하여 시각화하고, 주요 위험 회피 지점을 안내한다.
  7. 사용자가 안전 경로를 선택하면 길 안내 모드를 시작한다.
* **Subflows:**
  * **S1.** 가중치 계산: HW3 분석 기준에 따라 동일 위치 신고 횟수와 경찰청 범죄 통계 API를 가중치로 합산함.
  * **S2.** 실시간 업데이트: 경로 안내 중 경로상에 새로운 위험 신고가 접수되면 경로를 재탐색함.
* **Alternate / Exceptional Flows:**
  * **A1.** 도보/차량 모드 선택: 사용자의 이동 수단에 따라 위험 요소의 가중치를 다르게 적용(예: 도보는 골목길 위험도 중시)하여 경로를 제공함.
  * **E1.** 검색 결과 없음: 목적지가 존재하지 않거나 경로 생성이 불가능한 경우 "경로를 찾을 수 없습니다" 메시지를 출력함.
  * **E2.** 안전 경로 부재: 모든 경로의 위험도가 높을 경우, 주의 문구와 함께 가장 위험도가 낮은 대안을 제시함.

> **📌 UC_08: 실시간 위험 구역 진입 알림**
> * **Importance Level:** high
> * **Primary Actor:** 사용자
> * **Use Case Type:** Concrete, Essential

* **Brief Description:** 사용자가 앱을 사용 중이거나 백그라운드 상태에서 이동할 때, AI가 분석한 위험 구역 반경 내에 진입하면 푸시 알림이나 진동으로 즉시 경고를 전달한다.
* **Stakeholders and Interests:**
  * **사용자:** 스마트폰을 보고 있지 않더라도 위험 지역 진입을 즉시 인지하여 신변을 보호하고자 함.
  * **관리자:** 실시간 사고 예방 서비스를 통해 앱의 유용성과 신뢰도를 높임.
* **Trigger:** 사용자의 현재 위치 좌표가 사전에 정의된 위험 구역의 경계선 내로 진입함.
* **Relationships:**
  * **Association:** 사용자 (Actor)
  * **Include:** 위치 정보 실시간 추적
  * **Extend:** 위험 상세 정보 확인
  * **Generalization:** 없음.
* **Normal Flow of Events:**
  1. 시스템은 백그라운드에서 사용자의 GPS 위치를 주기적으로 모니터링한다.
  2. 현재 위치 반경 내에 서버로부터 동기화된 위험 구역 데이터를 대조한다.
  3. 사용자가 설정된 임계치(예: 위험 구역 50m 이내) 안으로 진입함을 감지한다.
  4. 시스템은 스마트폰에 푸시 알림, 진동, 혹은 음성 메시지를 송출한다.
  5. 사용자가 알림을 확인하면, 해당 구역의 위험 사유를 짧게 표시한다.
* **Subflows:**
  * **S1.** 알림 개인화: 사용자가 설정한 '위험도 수준(상, 중, 하)'에 따라 알림 민감도를 조절함.
  * **S2.** 저전력 모드: 배터리 효율을 위해 이동 속도에 따라 위치 수집 주기를 동적으로 변경함.
* **Alternate / Exceptional Flows:**
  * **A1.** 경로 안내 중 알림: 안전 경로 이용 중 예상치 못한 새로운 위험이 경로상에 발생할 경우 즉시 우회 알림을 보냄.
  * **E1.** GPS 신호 소실: 터널, 지하 등에서 위치 확인이 불가능할 경우 "위치 추적 일시 중단" 알림을 보냄.
  * **E2.** 알림 지연: HW2 리스크 관리 계획에 따라 네트워크 상태 불안정으로 인한 알림 지연 가능성을 사용자 가이드에 명시함.

> **📌 UC_09: 신고 신뢰도 평가**
> * **Importance Level:** medium
> * **Primary Actor:** 사용자
> * **Use Case Type:** Concrete, Essential

* **Brief Description:** 타인이 등록한 위험 신고 정보에 대해 실제 상황과 일치하는지 여부를 '도움됨' 또는 '부정확'으로 투표하여 시스템의 신뢰도 지수에 반영한다.
* **Stakeholders and Interests:**
  * **사용자:** 정확한 정보에 투표하여 커뮤니티의 안전 데이터 정제에 기여하고, 보상 포인트를 얻음.
  * **피신고자:** 본인의 신고가 인정을 받으면 신뢰 등급이 상승함.
  * **관리자:** 허위 신고를 자동으로 필터링하고 양질의 데이터를 확보함.
* **Trigger:** 사용자가 위험 요소 상세 정보를 확인한 후, 하단의 평가 버튼을 클릭함.
* **Relationships:**
  * **Association:** 사용자 (Actor)
  * **Include:** 없음.
  * **Extend:** 신고자 신뢰도 등급 업데이트, 허위 신고 자동 삭제
  * **Generalization:** 없음.
* **Normal Flow of Events:**
  1. 사용자가 특정 위험 요소의 상세 페이지에 접속한다.
  2. 시스템은 해당 정보에 대해 '실제 상황과 일치합니까?'라는 평가를 제공한다.
  3. 사용자가 '도움됨' 또는 '부정확' 버튼 중 하나를 클릭한다.
  4. 시스템은 사용자의 투표 결과를 서버로 전송한다.
  5. 시스템은 해당 게시물의 누적 신뢰도 점수를 실시간으로 재계산한다.
  6. 평가에 참여한 사용자에게 감사의 메시지와 함께 소정의 신뢰도 활동 포인트를 부여한다.
* **Subflows:**
  * **S1.** 가중치 적용: 평가를 내리는 사용자의 신뢰 등급이 높을수록 해당 투표가 결과에 미치는 영향력을 높게 설정함.
  * **S2.** 실시간 지도 연동: '부정확' 투표가 일정 비율 이상 쌓이면 지도상에서 해당 아이콘을 흐리게 표시하거나 검토 중 상태로 변경함.
* **Alternate / Exceptional Flows:**
  * **A1.** 투표 취소: 사용자가 마음을 바꿔 클릭했던 버튼을 다시 누르면 투표가 취소되고 점수가 원상복구됨.
  * **E1.** 본인 게시물 평가 불가: 자신이 올린 신고물에는 평가 버튼이 활성화되지 않도록 차단함.
  * **E2.** 중복 투표 방지: 동일한 사용자가 한 게시물에 여러 번 투표하는 것을 시스템적으로 제한함.

> **📌 UC_10: 내 활동 내역 및 등급 조회**

> * **Use Case Type:** Concrete, Essential

* **Stakeholders and Interests:**
  * **사용자:** 자신의 기여도를 확인하여 성취감을 얻고, 신뢰 등급에 따른 혜택이나 권한을 파악함.
  * **관리자:** 투명한 등급 공개를 통해 사용자가 자발적으로 양질의 데이터를 생산하도록 독려함.

* **Relationships:**
  * **Association:** 사용자 (Actor)
  * **Include:** 사용자 활동 데이터 로드
  * **Extend:** 신고 내용 상세 보기, 신고 내용 수정/삭제
  * **Generalization:** 없음.

  1. 사용자가 마이페이지 메뉴를 선택한다.
  2. 시스템은 서버에 해당 사용자의 누적 신고 수, 평가 수, 현재 신뢰도 점수를 요청한다.
  3. 서버는 계산된 신뢰도 등급과 포인트 정보를 응답한다.
  4. 시스템은 화면 상단에 현재 등급과 다음 등급까지 남은 점수를 시각화(게이지 바 등)하여 보여준다.
  5. 하단 리스트에는 사용자가 과거에 등록한 신고 내역들이 최신순으로 출력된다.
  6. 사용자는 리스트의 항목을 클릭하여 과거 신고의 처리 상태를 확인하거나 수정 화면으로 이동할 수 있다.
* **Subflows:**
  * **S1.** 등급 산정 로직: HW3 정책에 따라 [신고 횟수 + 타인으로부터 받은 도움됨 평가 - 허위 신고 페널티]를 합산하여 실시간 등급을 결정함.
  * **S2.** 비식별화 확인: 본인에게는 노출되지만 타인에게 공개될 시 마스킹되는 정보를 미리보기 형태로 제공함.
* **Alternate / Exceptional Flows:**
  * **A1.** 등급 하락 알림: 허위 신고 등으로 인해 신뢰 등급이 하락한 경우, 사유와 함께 경고 문구를 표시함.
  * **E1.** 활동 내역 없음: 신고 이력이 없는 신규 사용자의 경우 "첫 번째 신고를 통해 동네 안전에 기여해 보세요!"라는 가이드 문구를 출력함.
  * **E2.** 데이터 로딩 지연: 서버 응답이 늦어질 경우 UI를 보여주며 사용자 이탈을 방지함.

## 4. 인터페이스 분석

### 4.1 사용자 인터페이스
사용자 및 관리자가 시스템과 상호작요하기 위한 화면 및 UI 요소입니다. 모바일 앱 또는 웹 환경을 기준으로 구성됩니다.

**1) 일반 사용자 UI**
* **메인 지도 화면 (위험지도):** 현재 위치(GPS)를 중심으로 지도를 렌더링하며, 위험 데이터를 마커 또는 히트맵 형태로 시각화합니다.
* **위험 요소 신고 화면:** 현장 사진 등록, 위험 유형 선택, 상세 설명 입력 및 현재 GPS 위치 기반의 좌표 자동 입력 폼을 제공합니다.
* **안전 경로 탐색 화면:** 출발지와 목적지를 입력받고, 계산된 최적 우회 경로(안전 경로)를 지도상에 선으로 시각화하여 예상 소요 시간과 함께 출력합니다.
* **실시간 알림(Push) 화면:** 사용자 주변에 새로운 위험 요소가 접수되거나 검증되었을 때 팝업 및 푸시 알림 배너 형태로 경고를 제공합니다.

**2) 관리자 UI**
* **위험 데이터 대시보드:** 실시간으로 접수되는 신고 현황을 리스트 형태로 조회하고, 진척도를 관리하는 화면입니다.
* **신뢰도 및 회원 관리 화면:** 고의적인 허위 신고를 방지하기 위해 사용자별 신고 횟수와 평가(신뢰도 지수)를 모니터링하고 제재할 수 있는 인터페이스입니다.

### 4.2 시스템 간 연계
SafeMap 시스템이 외부 시스템 및 내부 모듈과 데이터를 주고받는 방식입니다.
* **통신 규격 및 보안:** 모든 시스템 간 데이터 송수신은 RESTful API 아키텍처를 기반으로 설계합니다. 보안 및 개인정보 보호를 위해 모든 통신 데이터는 HTTPS 프로토콜을 통해 암호화되어 전송됩니다.
* **외부 공공 API 연계:** 지역 내 CCTV 위치 정보, 경찰청 범죄 발생 통계 데이터 등을 주기적으로 호출(GET)하여 AI 위험도 분석의 가중치 데이터로 활용합니다.
* **지도 서비스 API:** 배경 지도 렌더링 및 기본 도로망 데이터 (Node/Link)를 가져오기 위해 연계합니다.
* **푸시 알림 서버 연계:** 실시간 위험 알림 전송을 위해 외부 푸시 알림 인프라와 연계합니다.

### 4.3 입/출력 인터페이스
시스템 내부 로직이 실제로 주고받는 데이터의 속성과 형태입니다.

**1) 지도 인터페이스**
* **입력:** 사용자 화면의 중심 좌표, 줌 레벨, 지도 세션 ID
* **출력:** 현재 화면 영역에 표시되어야 할 위험 데이터 마커 리스트. 히트맵 시각화를 위한 위험도 지수 배열

**2) 경로 인터페이스**
* **입력:** 경로 요청 ID, 출발지 좌표, 목적지 좌표
* **출력:** 위험지역을 회피하여 계산된 안전 경로의 좌표 배열 및 예상 소요시간

**3) 데이터 수집 인터페이스**
* **입력:** 사용자 ID, 신고 위치 좌표, 신고 텍스트 내용, 첨부 이미지 데이터, 신고 발생 시간
* **출력:** DB 저장 성공 여부, 실시간 알림 발송용 트리거 이벤트 데이터

## 5. 제약사항

### 1. 기술적 제약
* **AI 분석 데이터 확보:** 초기 서비스 단계에서 사용자 신고 데이터가 부족할 경우를 대비하여, 공공데이터를 활용한 초기 맵 구성이 필요합니다.
* **네트워크 의존성:** 실시간 지도 데이터 수집 및 알림 발송을 위해 인터넷 연결 환경에서만 모든 기능이 정상적으로 작동합니다.

### 2. 법적 및 보안 제약
* **위치 정보 필수 (위치 정보 활용):** 시스템의 핵심 기능인 위험 신고 및 실시간 알림을 위해 사용자의 GPS 위치 정보 수집에 대한 명시적인 동의 절차가 반드시 필요합니다.
* **개인정보 보호법 준수:** 사용자 신고 시 포함되는 사진 및 위치 정보 등이 노출될 경우를 대비하여 비식별화 처리 또는 관리 정책을 수립해야 합니다.

### 3. 운영적 제약
* **허위 신고 방지:** 특정 지역의 평판을 저해할 목적으로 행해지는 악의적인 허위 신고를 걸러내기 위한 알고리즘이 설계에 반영되어야 합니다.
* **실시간 알림 지연:** 네트워크 상태에 따라 실시간 위험 알림 푸시 메시지가 지연될 수 있음을 사용자 가이드에 명시해야 합니다.

### 4. 자원 제약
* **개발 기간 엄수:** 2026년 6월 16일까지 3개월이라는 한정된 기간 내에 시스템 구현을 완료해야 합니다.

* 문서와 CRC 카드 내용에서 도출된 주요 유스케이스 목록 및 인터페이스 역할을 하는 CRC(위험지도, 안전경로)의 세부정보

### 7.3 분석 보조 자료
* **타 서비스 벤치마킹 자료:** 문서에 언급된 미국 안전 앱 'Citizen(시티즌)'의 주요 기능을 분석하고 SafeMap에 어떻게 적용할지 검토한 내용
* **공공데이터 API 검토 자료:** 초기 데이터 부족 문제를 해결하기 위해 조사하신 지자체 CCTV 위치 정보 및 경찰청 5대 범죄 발생 통계 API의 활용 가능성 검토 내역
* **위험 분석 자료:** 개발 방법론으로 '나선형 모델(Spiral Model)'을 채택하며 프로젝트 초기에 진행하셨던 리스크 식별 및 대안 평가 자료
* **팀 회의록 및 요구사항 도출 결과:** 팀원들 (김하양, 김파랑, 김초록)의 요구사항을 추려낸 회의 요약본이나 마인드맵
