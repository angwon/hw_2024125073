# [세이프맵] 동네 위험지도 자동 생성 서비스
> 사용자 신고 기반 위험 정보 수집 및 AI 분석을 통한 안전 경로 추천 서비스

---

## 1. 프로젝트 개요
### 1.1 문서의 목적 및 범위
본 프로젝트 계획서는 동네 위험지도 자동 생성 및 안전 경로 추천과 실시간 알림 서비스에 대한 전반적인 내용을 정리하여 프로젝트를 진행하는 데 있어 각 세부 사항의 지표로 삼는 것을 목적으로 합니다.

### 1.2 프로젝트 정의
사용자 신고 데이터를 기반으로 지역 내 위험 정보를 수집하고 **AI 분석**을 통해 동네 안전 지도를 생성하여 **최적의 안전 경로 추천 및 실시간 알림**을 제공하는 스마트 안전 서비스를 개발합니다.

### 1.3 프로젝트 목표
- 사용자 신고 데이터 기반 위험 요소 시각화
- AI 분석을 통한 최적 안전 경로 제공
- 실시간 위험 지역 접근 알림 서비스 구축

### 1.4 주요 기능 (Features)
| ID | 기능명 | 상세 설명 |
|:---:|:---:|:---|
| **F01** | 위험 요소 신고 | 사용자가 GPS를 활용해 위치, 사진, 위험 유형을 직접 신고 |
| **F02** | 위험 지도 자동 생성 | 신고된 정보를 바탕으로 마커 표시 및 위험도별 색상 구분 지도 생성 |
| **F03** | AI 기반 위험도 분석 | 신고 횟수, 유형, 최근 신고 시간 등을 분석하여 객관적 위험 수치 산출 |
| **F04** | 안전 경로 추천 | 목적지 설정 시 위험 지역을 회피하는 최적의 안전 이동 경로 제공 |
| **F05** | 실시간 위험 알림 | 위험 지역 접근 시 스마트폰 푸시 알림을 통해 사전 주의 정보 제공 |

## 2. 개발 계획
### 2.1 개발 방법론
* **개발 모델:** **나선형 모델(Spiral Model)** 응용 (리스크 최소화 및 반복적 요구사항 반영)
* **설계 방법론:** **컴포넌트 기반 설계(CBD)** (GPS, 알림 등 기능을 독립적 부품 형태로 개발)

### 2.2 WBS (Work Breakdown Structure)
1. **요구사항 분석 및 기획:** 유사 시스템 분석, 상세 정의, 위험도 산출, 기능 명세서
2. **시스템 설계:** DB 설계, AI 모델링, UI/UX 설계, 시스템 구조 설계
3. **기능 구현 및 개발:** 신고 모듈, AI 분석, 실시간 지도, 경로 추천, 푸시 알림
4. **테스트 및 마무리:** 단위/통합/성능 테스트, 문서화, 최종 발표

### 2.3 개발 일정
* **전체 기간:** 2026.03.16 ~ 2026.06.16 (3개월)
* **스프린트 운영:** 2주 단위의 Sprint 구성 (총 5개 스프린트)
